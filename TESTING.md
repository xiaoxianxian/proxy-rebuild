# 正负例清单

## 正例（预期行为）

### 概览页
- 页面加载后 3/3 运行中（或实际运行数量）
- 已安装代理显示为 3
- 代理状态列表显示 Codex Proxy、Hermes Proxy、Cursor Proxy
- 每个代理卡片显示运行状态（绿色/红色圆点）、端口号、健康状态
- 运行中的代理显示"停止"按钮，未运行的显示"启动"按钮

### 代理配置页
- 点击配置按钮跳转到对应代理的配置页
- 配置页显示基本信息（运行状态、端口）
- 详细信息显示当前配置、Provider 状态、切换历史
- 刷新按钮触发 API 数据重新加载

### 日志页
- 默认显示"暂无日志，等待请求..."
- 选择"全部代理"后点击刷新，显示所有代理的日志
- 选择特定代理后刷新，只显示该代理的日志
- 日志中的特殊字符被正确转义

### 服务管理
- `manage.sh start` 启动所有服务
- `manage.sh stop` 停止所有服务
- `manage.sh status` 显示各服务状态
- `manage.sh logs all` 显示所有日志
- 代理进程不存在时，管理器通过端口检测识别为运行中

## 负例（不应出现的行为）

### 概览页
- ~~页面显示"-"和"正在加载..."且不再变化~~（已修复：JS 语法错误）
- ~~代理状态为空时显示"暂无代理"而非"正在加载..."~~（已修复）
- 刷新页面后仍然卡在加载状态
- 统计数字与实际运行代理数不符

### 代理配置页
- ~~点击配置按钮后页面空白~~（已修复）
- 代理详情数据加载失败时无错误提示
- 切换历史显示过时或重复的记录
- Provider 状态与实际不符

### 日志页
- 日志中包含未转义的 HTML/JS 代码（XSS 风险）
- 选择"全部代理"时报错
- 日志刷新后显示之前已清除的内容

### 服务管理
- 重复启动已运行的代理
- 停止不存在的代理时报错
- 端口冲突时未提示用户
- 开机自启设置后重启不生效
